# Hackathon checklist

## Before the hackathon

- [ ] Verify local setup with `make install`
- [ ] Copy and verify environment variables
- [ ] Verify mock AI mode without API keys
- [ ] Verify frontend and backend startup with `make dev`
- [ ] Verify Vercel and Railway/Render deployment
- [ ] Verify `make reset`
- [ ] Rehearse the Maya Thompson workflow

## During the hackathon

- [ ] Validate the workflow with clinicians
- [ ] Replace generic copy with the selected use case
- [ ] Adapt fictional patient data
- [ ] Update and evaluate AI prompts
- [ ] Build the core differentiating feature
- [ ] Test the complete demo
- [ ] Deploy frontend and backend
- [ ] Practice the pitch

## Final two hours

- [ ] Freeze nonessential features
- [ ] Fix demo blockers only
- [ ] Improve critical loading and error states
- [ ] Reset demo data
- [ ] Record a backup demo
- [ ] Practice the presentation
